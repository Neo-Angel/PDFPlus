


PP~ : 모든 글로발 범위의 이름들(Class, Constant)은 PP라는 접두사를 가진다. 이는 [P]df [P]lus를 의미한다.
Namespace 관련: PDF+는 네임 스페이스를 사용하지 않는다. PP접두사를 씀으로서 어느 코드에 섞여있어도 Pdf+ 용 클래스라는 것을 쉽게 알 수 있다.
public members : 대부분의 멤버변수, 함수들은 public으로 정의한다. 서드파티 개발자들에게 코딩의 자유도를 높이기 위함이다.

Naming Rule : 
	- 모든 클레스는 PP로 시작한다. 그 뒤에 붙는 문자는 접두사를 뺀 슈퍼클래스 명의 첫글자이다.
		예를 들어 Path Element 클래스의 경우 슈퍼 클래스가 PPElement이므로 PPEPath가 된다. 
	- struct, enum, classs 명들은 모두 같은 룰을 적용한다.
	- 멤버 변수들은 모두 '_'로 시작한다.
	- protected 멤버함수들은 소문자로 시작한다.
	- protected 멤버함수를 제외한 모든 멤버함수는 대문자로 시작한다.
	- 일반 전역 함수들은 'PP'로 시작한다.
	- 일반 전역 상수들은 'PPXX_'로 시작하며 'XX'는 타입을 의미한다. 모두 대문자로 이루어진다.
	- enum 상수들은 'PPXX_' 로 시작하며 'XX'는 타입을 의미한다. 뒤에 이어지는 문자는 Word단위로 대/소문자를 혼합한다.
	- struct 멤버들은 모두 소문자로 시작한다.
	- 함수 내에 로컬 변수들과 함수 파라미터들은 모두 소문자로 시작하고 단어간에는 '_'로 연결한다.
	- 함수 파라미터는 필요한 경우 왼쪽이 소스고 오른쪽이 타겟으로 한다. 


* 멤버 변수 타입

리퍼런스 : 복사 되지 않으며 주로 자신의 소속과 관련된 상위 객체나 연관된 객체를 참조하기 위한 멤버 포인트
		ex) *_dociment, *_parent
속성객체 : 속성 객체들. 각 객체별로 가지는 속성 객체들
		ex) _color, _name 
기능객체 : 부모 객체를 생성할 때 같이 생성되는 객체로 부모객체와 생명주기를 같이한다. 주로 기능성 객체.
		ex) _Parser, _Dict, _List



* 멤버함수의 타입들
	- Set... : 인수가 객체일 경우 포인터만 넘겨줌
	- Put... : 인수가 객체일 경우 객체를 복사함.
	- Write... : 인수가 객체일 경우 객체를 복사하고 관련 리퍼런스들도 리시버 기준으로 전환됨. 
	- Get... : 인수를 통해서 값을 받아옴
	- 멤버변수명('_'가 빠지고 대문자로 시작하는)함수() : 멤버변수 값을 리턴함

* 함수 규칙
	- 파라미터는 의미상의 순서대로 배치한다.
	- 보통은 source가 먼저 나오고 target이 뒤에 나온다.

* Class 설명

- PPBase : PDF Plus 모든 클래스의 기반이 되는 클래스.
- PPToken : 저수준 자료 구조를 위한 기반 클래스

- PPElement : 서드 파티 개발자들을 위한 API의 기반 클래스
