


PP~ : 모든 글로발 범위의 이름들(Class, Constant)은 PP라는 접두사를 가집니다. 이는 [P]df [P]lus를 의미합니다.
Namespace 관련: PDF+는 네임 스페이스를 사용하지 않습니다. PP접두사를 씀으로서 어느 코드에 섞여있어도 Pdf+ 용 클래스라는 것을 쉽게 알 수 있습니다.
public members : 모든 멤버변수, 함수들은 모두 public으로 정의했습니다. 구분하기 귀찮았던 것도 있지만 서드파티 개발자들에게 코딩의 자유도를 높이고자 하는 의도도 있습니다.

Naming Rule : 모든 클레슨는 PP로 시작합니다. 그 뒤에 붙는 문자는 접두사를 뺀 슈퍼클래스 명의 첫글자입니다.
		예를 들어 Path Element 클래스의 경우 슈퍼 클래스가 PPElement이므로 PPEPath가 됩니다. 
		익숙해지면 PPEPath 를 처음 보더라도 '아~ PPElement의 서브 클래스구나~' 라고 알 수가 있습니다.




* 멤버 변수의 세가지 타입

리퍼런스 : 복사 되지 않으며 주로 자신의 소속과 관련된 상위 객체나 연관된 객체를 참조하기 위한 멤버 포인트
		ex) *_dociment, *_parent
속성객체 : 속성 객체들. 각 객체별로 가지는 속성 객체들. 주로 포인터로 표현됨
		ex) *_color, *_name 
고정객체 : 부모 객체를 생성할 때 같이 생성되는 객체로 부모객체와 생명주기를 같이한다. 주로 기능성 객체.
		ex) _parser, _dict, _list


* PPToken : 저수준 자료 구조를 위한 클래스

* PPElement : 개발자 API 를 위한 클래스

* 멤버함수의 타입들
	- Set... : 인수가 객체일 경우 포인터만 넘겨줌
	- Write... : 인수가 객체일 경우 객체를 복사하고 관련 리퍼런스들도 리시버 기준으로 전환됨.
